问题1 - 特征观察¶

如前文所述，本项目中我们关注的是其中三个值:'RM'、'LSTAT' 和'PTRATIO'，对每一个数据点:

  • 'RM' 是该地区中每个房屋的平均房间数量；
  • 'LSTAT' 是指该地区有多少百分比的房东属于是低收入阶层（有工作但收入微薄）；
  • 'PTRATIO' 是该地区的中学和小学里，学生和老师的数目比（学生/老师）。

凭直觉，上述三个特征中对每一个来说，你认为增大该特征的数值，'MEDV'的值会是增大还是减小呢？每一个答案都需要你给出理由。

提示：你预期一个'RM' 值是6的房屋跟'RM' 值是7的房屋相比，价值更高还是更低呢？

回答:

  1. 增大RM值应该会使MEDV值上升。因为一般来讲，房间数量越多说明房屋面积越大，房屋面积大的往往总价值会比较高。
  2. 增大LSTAT会使MEDV值下降。这是因为低收入阶层比例增大时，该地区的整体环境（包括人文、经济、卫生）会相对下降，如此就不能吸引愿意出更多钱购买改善生活环境房屋的购房者，所以房屋价格会下降。
  3. 增大PTRATIO会使MEDV值下降。这是因为当学生/老师的数字增大时，说明每个学生分配到的教师数量减少，间接说明教育水平下降，那么在教育水平较低的地区，房屋价格也会较低。

练习：定义衡量标准¶

如果不能对模型的训练和测试的表现进行量化地评估，我们就很难衡量模型的好坏。通常我们会定义一些衡量标准，这些标准可以通过对某些误差或者拟合程度的计算来得到。在这个项目中，你将通过运算决定系数 R2 来量化模型的表现。模型的决定系数是回归分析中十分常用的统计信息，经常被当作衡量模型预测能力好坏的标准。

R2的数值范围从0至1，表示目标变量的预测值和实际值之间的相关程度平方的百分比。一个模型的R2 值为0还不如直接用平均值来预测效果好；而一个R2 值为1的模型则可以对目标变量进行完美的预测。从0至1之间的数值，则表示该模型中目标变量中有百分之多少能够用特征来解释。模型也可能出现负值的R2，这种情况下模型所做预测有时会比直接计算目标变量的平均值差很多。

在下方代码的 performance_metric 函数中，你要实现：

  • 使用 sklearn.metrics 中的 r2_score 来计算 y_true 和 y_predict的R2值，作为对其表现的评判。
  • 将他们的表现评分储存到score变量中。

问题 8- 交叉验证¶

什么是K折交叉验证法（k-fold cross-validation）？优化模型时，使用这种方法对网格搜索有什么好处？

提示： 跟为何需要一组测试集的原因差不多，网格搜索时如果不使用交叉验证会有什么问题？

答案: 将数据集按量平均分为K份，用其中1份数据作为测试数据，另外的K-1份数据作为训练数据，如此法用数据集训练K次模型，每次都用不同的1份测试数据验证模型效果，此为K折验证法。使用这种方法相较于网格搜索能更好的得知模型对未见过的数据预测的能力，因为网格搜索用全部数据训练，没有使用为参与训练的数据进行测试。

# TODO: Import 'make_scorer', 'DecisionTreeRegressor', and 'GridSearchCV'
from sklearn.tree import DecisionTreeRegressor
from sklearn.metrics import make_scorer
from sklearn.model_selection import GridSearchCV
#from sklearn.cross import ShuffleSplit


def fit_model(X, y):
    """ Performs grid search over the 'max_depth' parameter for a 
        decision tree regressor trained on the input data [X, y]. """
    
    # Create cross-validation sets from the training data
    #因为我本地使用的scikit-learn版本是0.18.1，其中的ShuffleSplit函数本文档代码不符，所以做了如下修改
    #cv_sets = ShuffleSplit(X.shape[0], n_splits = 10, test_size = 0.20, random_state = 0)
    rs = ShuffleSplit(n_splits = 10, test_size = 0.20, random_state = 0)
    #use scikit-learn 0.18.1 version ShuffleSplit
    cv_sets = rs.split(X)
    #Just for debugging
    #print type(X),type(cv_sets),cv_sets


    # TODO: Create a decision tree regressor object
    regressor = DecisionTreeRegressor(random_state=0)

    # TODO: Create a dictionary for the parameter 'max_depth' with a range from 1 to 10
    params = {'max_depth':[x for x in range(1,10)]}

    # TODO: Transform 'performance_metric' into a scoring function using 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric)

    # TODO: Create the grid search object
    grid = GridSearchCV(regressor, params, scoring_fnc, cv=cv_sets)

    # Fit the grid search object to the data to compute the optimal model
    grid = grid.fit(X, y)

    # Return the optimal model after fitting the data
    return grid.best_estimator_

问题 11 - 实用性探讨¶

简单地讨论一下你建构的模型能否在现实世界中使用？

提示： 回答几个问题：

  • 1978年所采集的数据，在今天是否仍然适用？
  • 数据中呈现的特征是否足够描述一个房屋？
  • 模型是否足够健壮来保证预测的一致性？
  • 在波士顿这样的大都市采集的数据，能否应用在其它乡镇地区？

答案:

  1. 1978年所采集的数据在今天已经不适用，因为房屋周边的环境发生了较大的变化，例如教育资源和贫穷指数
  2. 数据中呈现的特征作为评估房价来说少了些，还有其他一些重要的特征，例如：房屋年龄、交通条件、距离商业区的距离等
  3. 模型的健壮性尚且不足，因为当前特征较少，训练数据集也较小，使用模型预测未知数据时会出现较大的不一致性
  4. 在波士顿采集的数据不适用于其它乡镇地区。举例来说，乡镇房屋的面积普遍较大，所以增大房屋面积不会大幅提高房屋价格。
